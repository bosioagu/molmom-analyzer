Guía de deploy

Cómo subir a GitHub Pages

5 pasos · 5 minutos · gratis para siempre

1
Crear repositorio en GitHub
Entrá a github.com → New repository → Nombre: molmom-analyzer → Public → Create
2
Subir el archivo
En el repo → Add file → Upload files → arrastrá el archivo analizador-producto.html → Commit changes
3
Renombrarlo a index.html
En el repo hacé clic en el archivo → el ícono del lápiz (edit) → cambiá el nombre a index.html → Commit
4
Activar GitHub Pages
Settings → Pages → Source: Deploy from branch → Branch: main → / (root) → Save
✓
¡Listo! Tu URL queda:
https://TU-USUARIO.github.io/molmom-analyzer/
Compartila con los 4 del equipo. Funciona desde cualquier celular o PC.
Por qué funciona hosteado y no como archivo local
Los navegadores bloquean llamadas a APIs externas desde archivos file:// por seguridad (política CORS). Cuando la app está en un servidor real como GitHub Pages, las llamadas a api.mercadolibre.com pasan sin problema porque ML permite requests desde cualquier origen en su API pública.
